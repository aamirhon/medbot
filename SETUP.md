## 1. Архитектура

```
┌─────────────┐                       ┌──────────────┐
│   Клиент    │ ─── /start, ИНН ───→  │  Telegram    │
│  (Telegram) │                       │     Bot      │
└─────────────┘                       └──────┬───────┘
                                             │
                                             │ HTTP
                                             ▼
                                      ┌──────────────┐
                                      │   1С HTTP    │
                                      │   сервис     │
                                      └──────┬───────┘
                                             │
                                      ┌──────▼───────┐
                                      │ 1С документы │
                                      │ (контрагенты,│
                                      │  заказы,     │
                                      │  счета)      │
                                      └──────────────┘
```

Поток заказа:

1. Клиент в боте вводит ИНН
2. Бот спрашивает у 1С — есть такой контрагент?
3. 1С возвращает name, tier, address
4. Бот регистрирует пользователя
5. Клиент выбирает товары → корзина → оформление
6. Бот создаёт заказ в БД и отправляет в 1С
7. 1С генерирует счёт + договор → возвращает PDF URLs
8. Бот пересылает PDF клиенту в чат
9. Клиент платит банковским переводом по реквизитам в счёте
10. Бот раз в 10 минут проверяет в 1С — оплачено ли?
11. После подтверждения оплаты статус меняется → клиент получает уведомление

## 2. Подготовка

### Что нужно получить заранее

| Что | Где | Зачем |
|---|---|---|
| Bot Token | @BotFather в Telegram | для работы бота |
| 1С база с контрагентами | у вашего 1С-программиста | источник данных |
| VPS (2 CPU, 4 GB RAM) | Hetzner, Contabo, или локальный | где запускать бота |
| Домен | reg.uz, namecheap | для HTTPS и WebApp |
| Публичная папка для PDF | веб-сервер на 1С | чтобы бот мог скачать счёт |

### Получение Bot Token

1. Откройте Telegram, найдите @BotFather
2. Отправьте `/newbot`
3. Введите имя бота: `Медбот для клиник`
4. Введите username: `my_medbot_bot` (должен заканчиваться на `bot`)
5. Получите токен вида `1234567890:ABCdefGhIJK...`
6. Сохраните в `.env` как `BOT_TOKEN=...`

### Получение Telegram ID администратора

1. Найдите @userinfobot в Telegram
2. Отправьте `/start`
3. Бот пришлёт ваш ID (например, `123456789`)
4. Запишите в `.env` как `ADMIN_TELEGRAM_IDS=123456789`

## 3. Настройка 1С

### Что должно быть в 1С

Чтобы бот работал, в 1С должны существовать:

1. Справочник `Контрагенты` с реквизитами:
   - `ИНН` (обязательно)
   - `УровеньКлиента` (перечисление: VIP / Стандартный / Ограниченный)
   - `ТипКлиента` (перечисление: Госучреждение / ЧастнаяКлиника)
   - Стандартные: Наименование, КонтактныйТелефон, ЮридическийАдрес

2. Справочник `Номенклатура` со штатными реквизитами:
   - `Артикул`, `Наименование`, `Родитель` (категория)
   - Цены через регистр сведений `ЦеныНоменклатуры`

3. Документы:
   - `ЗаказПокупателя` — основной документ заказа
   - `СчётНаОплату` — генерируется на основании заказа
   - Печатные формы для счёта и договора

### Создание HTTP-сервиса

В Конфигураторе:

1. Откройте конфигурацию (Конфигурация → Открыть конфигурацию)
2. Раздел `Общие` → `HTTP-сервисы` → `Добавить`
3. Задайте имя `BotIntegration`, корневой URL `bot`
4. Создайте шаблоны URL согласно файлу `onec_examples/HTTPService.bsl`
5. Перенесите код функций обработчиков из этого файла
6. Сохраните и обновите конфигурацию (`F7`)

### Публикация HTTP-сервиса на веб-сервере

Для Apache (Linux):
```bash
# В Конфигураторе: Администрирование → Публикация на веб-сервере
# Или вручную через webinst:
sudo /opt/1cv8/x86_64/8.3.XX/webinst -apache24 \
    -wsdir bot \
    -dir /var/www/html/bot \
    -connstr "Srvr=localhost;Ref=YourBase" \
    -confPath /etc/apache2/apache2.conf
```

Для IIS (Windows): аналогично через `webinst.exe -iis ...`

После публикации сервис доступен по URL вида:
`http://your-1c-server/bot/hs/clients/check?inn=123456789`

### Создание пользователя для интеграции

В режиме Предприятия:
1. Администрирование → Пользователи → Добавить
2. Имя: `integration_bot`
3. Пароль: задайте сложный
4. Аутентификация 1С: включить
5. Доступ: только чтение справочников + создание ЗаказПокупателя/СчётНаОплату

### Настройка публичной папки для PDF

PDF счетов должны быть доступны по HTTPS. Варианты:

**Вариант A — через тот же веб-сервер:**
```nginx
# Nginx конфиг
location /bot_files/ {
    alias /var/www/bot_files/;
    autoindex off;
}
```
В коде 1С функция `СохранитьПечатнуюФорму` сохраняет PDF в эту папку.

**Вариант B — через S3:**
1С загружает PDF в Yandex Object Storage / AWS S3, возвращает signed URL.

### Тестирование HTTP-сервиса

Из браузера или curl:
```bash
curl -u integration_bot:password \
  "http://your-1c-server/bot/hs/clients/check?inn=123456789"
```
Ожидаемый ответ:
```json
{"found": true, "one_c_id": "...", "name": "Tashkent Clinic", "tier": "vip"}
```

## 4. Настройка сервера

### Аренда VPS

Рекомендуется:
- Hetzner CX22 (~€5/мес): 2 vCPU, 4 GB RAM, 40 GB SSD
- Contabo VPS S (~€7/мес): 4 vCPU, 8 GB RAM, 50 GB NVMe
- Любой Ubuntu 22.04 LTS

### Первичная настройка

```bash
ssh root@your-vps-ip

# Обновление
apt update && apt upgrade -y

# Создаём пользователя
adduser medbot
usermod -aG sudo medbot

# Docker
curl -fsSL https://get.docker.com | sh
usermod -aG docker medbot

# Файрвол
ufw allow OpenSSH
ufw allow 80
ufw allow 443
ufw enable
```

### Получение HTTPS-сертификата (если нужен webhook)

```bash
apt install certbot
certbot certonly --standalone -d bot.example.uz
# Сертификаты появятся в /etc/letsencrypt/live/bot.example.uz/
```

## 5. Развёртывание бота

### Загрузка кода

```bash
su - medbot
cd ~
git clone https://github.com/yourorg/medbot.git
# Или: scp -r medbot medbot@your-vps:~/
cd medbot
```

### Настройка переменных окружения

```bash
cp .env.example .env
nano .env
```

Заполните:
```ini
BOT_TOKEN=1234567890:ABCdefGhIJK...
ADMIN_TELEGRAM_IDS=123456789

DATABASE_URL=postgresql+asyncpg://medbot:strong_password@postgres:5432/medbot
REDIS_URL=redis://redis:6379/0

ONE_C_URL=http://1c-server.example.uz/YourBase
ONE_C_USER=integration_bot
ONE_C_PASSWORD=your_1c_password
```

В `docker-compose.yml` поменяйте пароль PostgreSQL на сильный и совпадающий с `DATABASE_URL`.

### Запуск

```bash
docker compose up -d --build
docker compose logs -f bot
```

Ожидаемый вывод:
```
[INFO] Bot started: @my_medbot_bot
[INFO] [sync products] {'created': 0, 'updated': 0, ...}
```

Если первая синхронизация прошла без ошибок — связь с 1С работает.

## 6. Наполнение данных

### Загрузка каталога из 1С

Каталог подтягивается автоматически каждые 15 минут. Чтобы запустить вручную:

```bash
docker compose exec bot python -c \
  "import asyncio; from services.sync import sync_products, make_client; \
   asyncio.run(sync_products(make_client()))"
```

После первой синхронизации в БД появятся товары и категории.

### Проверка через psql

```bash
docker compose exec postgres psql -U medbot -d medbot
```
```sql
SELECT count(*) FROM products WHERE is_active;
SELECT name, count(*) FROM categories GROUP BY name LIMIT 10;
```

## 7. Настройка лимитов

Лимиты задаются в таблице `tier_limits`. Рекомендуемые значения для трёх уровней:

```sql
-- Дефолтные лимиты (применяются если для категории нет специального правила)
INSERT INTO tier_limits (id, tier, category_id, max_qty_per_order, max_qty_per_month, max_amount_per_month) VALUES
  (gen_random_uuid()::text, 'vip',      NULL, 10000, 50000, 1000000000),
  (gen_random_uuid()::text, 'standard', NULL, 1000,  5000,  100000000),
  (gen_random_uuid()::text, 'limited',  NULL, 100,   500,   10000000);
```

Для конкретной категории (например, "Хирургические инструменты"):
```sql
INSERT INTO tier_limits (id, tier, category_id, max_qty_per_order, max_qty_per_month, max_amount_per_month)
SELECT gen_random_uuid()::text, 'limited', c.id, 10, 50, 5000000
FROM categories c WHERE c.name = 'Хирургические инструменты';
```

В будущем сделайте админ-панель в боте для редактирования этих значений.

## 8. Проверка работы

### Сценарий полного теста

1. Откройте бота в Telegram → нажмите `/start`
2. Введите ИНН существующего контрагента из 1С
3. Подтвердите регистрацию → должно показать ваш уровень
4. Откройте каталог → выберите категорию → товар
5. Добавьте 5 штук в корзину
6. Откройте корзину → "Оформить заказ" → введите комментарий
7. Проверьте: должно прийти подтверждение + PDF счёта + PDF договора
8. В 1С проверьте: появился ли документ ЗаказПокупателя и СчётНаОплату
9. Проведите оплату в 1С (вручную или загрузите выписку)
10. Через 10 минут бот должен прислать уведомление "Оплата получена"

### Чек-лист отладки

| Проблема | Решение |
|---|---|
| ИНН не находится | Проверьте curl-ом запрос к 1С напрямую |
| PDF не открывается | Проверьте, доступен ли URL по HTTPS |
| Бот не отвечает | `docker compose logs bot` — ищите ошибки |
| Каталог пустой | Проверьте `sync_logs` — есть ли записи с status=ok |
| Лимиты не работают | Убедитесь что строка в `tier_limits` есть для нужного tier |

## 9. Что дальше

После успешного запуска MVP добавьте:

1. **Telegram Mini App** (React) — для удобной навигации по 700+ товарам с поиском, фильтрами, фото
2. **Админ-панель** в боте — управление лимитами, верификация клиентов, рассылки
3. **Webhook вместо polling** — для production-надёжности
4. **Sentry** — для мониторинга ошибок
5. **Бэкапы PostgreSQL** — автоматический дамп каждые 6 часов на S3
6. **CI/CD через GitHub Actions** — автодеплой при push в main
7. **Healthcheck для 1С** — алерт если 1С недоступна > 30 минут
