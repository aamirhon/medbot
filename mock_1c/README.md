# Mock 1С — заглушка для локальной разработки

Эмулирует HTTP-сервис 1С пока настоящая 1С не готова. Позволяет запускать и тестировать бота полностью локально.

## Запуск

```bash
pip install fastapi uvicorn
uvicorn mock_1c.server:app --host 0.0.0.0 --port 8001 --reload
```

Откройте http://localhost:8001 — увидите список доступных эндпоинтов.

## В .env бота укажите

```
ONE_C_URL=http://localhost:8001
ONE_C_USER=mock
ONE_C_PASSWORD=mock
```

## Тестовые ИНН

| ИНН | Организация | Уровень |
|---|---|---|
| `300123456` | Городская клиническая больница №1 | VIP |
| `200987654` | Медцентр «Здоровье+» | Стандартный |
| `300555111` | Частная стоматология «Дент» | Ограниченный |

## Тестирование полного цикла заказа

1. В боте: `/start` → введите ИНН `300123456` → подтвердите
2. Откройте каталог → выберите товар → добавьте в корзину
3. Откройте корзину → оформите заказ
4. Бот пришлёт PDF счёта и PDF договора (тестовый PDF)
5. Чтобы эмулировать оплату — узнайте `one_c_id` заказа:
   ```bash
   curl http://localhost:8001/_admin/orders | jq
   ```
6. Пометьте заказ как оплаченный:
   ```bash
   curl -X POST http://localhost:8001/_admin/orders/<one_c_id>/mark-paid
   ```
7. Через 10 минут (или сразу если ускорить интервал в `.env`) бот пришлёт уведомление об оплате

## Эмуляция отгрузки

```bash
curl -X POST http://localhost:8001/_admin/orders/<one_c_id>/ship
```

После следующего цикла `orders_loop` бот пришлёт уведомление о смене статуса.
